1. 项目背景与痛点

1.1 业务背景

在传统的电商搜索推荐体系中，供给侧的更新往往依赖于历史数据反馈（如销量、点击率）。这种模式存在显著的滞后性：
•   新品冷启动难：如“拉布布（Labubu）世界杯限定款”，由于缺乏历史销量数据，系统无法识别其潜在热度，导致曝光不足。

•   热点反应慢：如“突发降温”或“世界杯开幕”，运营需人工感知并配置资源，通常存在 2-3 天的空窗期。

•   场景理解浅：传统算法难以理解“世界杯”与“护肝片”、“熬夜能量”之间的深层因果关联。

1.2 解决方案

本项目旨在通过引入 GLM 大模型，重构传统供给链路。将 LLM 作为系统的 “感知器官” 和 “推理大脑”，实现对外部热点的毫秒级响应，自动构建时空场景并完成商品匹配。

3. 核心功能模块

3.1 热点感知模块 (Hot Perception)

•   功能：自动抓取全网高热度话题。

•   输入：百度热搜、微博热搜实时榜单。

•   输出：经过清洗的新闻标题列表（Top 10）。

3.2 场景挖掘模块 (Scene Mining) —— 核心

•   功能：利用 LLM 将非结构化的新闻转化为结构化的购物场景。

•   输入：“2026世界杯今日开幕，梅西领衔阿根廷队...”

•   输出 (JSON Schema)：
    {
      "scene_name": "世界杯球迷零食补给站",
      "scene_type": "赛事/热点",
      "trigger_event": "世界杯小组赛开赛",
      "temporal_scope": "2026-06-14 至 2026-07-15",
      "geo_scope": "全国",
      "user_intent": "看球时需要边吃边喝，避免饿肚子影响观赛体验",
      "potential_keywords": ["啤酒", "薯片", "小龙虾", "护肝片", "熬夜能量"],
      "target_population": "男性球迷、聚会人群"
    }
    
•   Prompt 策略：采用 JSON Mode 强制约束输出格式，确保下游系统可读。

3.3 商品匹配模块 (Product Matching)

•   功能：基于场景关键词，自动关联商品库。

•   逻辑：

    1.  解析 potential_keywords。
    2.  在模拟数据库中检索包含关键词的商品。
    3.  按相关性排序，返回 Top 3 商品。
•   示例：

    ◦   场景关键词：["啤酒", "世界杯"]

    ◦   匹配商品：青岛啤酒经典10度整箱装

5. 预期效果与演示价值

5.1 功能效果

1.  零人工干预：从热点爆发到场景生成，全过程无需人工配置。
2.  毫秒级响应：相比传统 T+3 的运营周期，实现 T+0 实时响应。
3.  因果推理：不仅识别“世界杯”，还能推导出“护肝片”等隐性需求。

5.2 面试演示价值

在面试中，本 Demo 将用于证明：
•   AI-Native 思维：不是“在原系统上加 AI”，而是“用 AI 重构系统”。

•   技术落地能力：熟练掌握 LLM 的 Function Call/JSON 输出控制。

•   业务 Owner 意识：关注 GMV 转化、库存匹配和业务闭环，而非单纯的技术炫技。
